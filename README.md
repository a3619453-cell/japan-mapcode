# 日本地點代碼查詢

純前端靜態網頁，可直接放上 GitHub Pages（無需後端、無需 build）。

## 檔案
- `index.html` — 主程式（UI + 代碼演算法）
- `zonecoords.js` — 日本 1163 個 zone 的邊界座標資料（沿用先前逆向工程成果）

兩個檔案需放在同一層目錄。

## 功能
- **地名搜尋**：可切換 OSM（Nominatim，免費免金鑰）與 Google。OSM 每筆結果附「在 Google 地圖開啟」連結。
- **經緯度**：直接輸入，或貼上「35.6586, 139.7454」自動拆分。
- **點地圖選位置**：啟用點選模式後於地圖按一下即算出代碼。
- **反查**：輸入代碼（mapcode 相容格式）定位到網格中心（誤差約 ±15 公尺）。
- 查詢紀錄存於瀏覽器 localStorage。
- iOS 26 風格 Liquid Glass 介面：邊緣折射透鏡（中央清晰、邊緣放大彎曲背景）、鏡面高光、隨手機傾斜（陀螺儀）或滑鼠移動而流動的反光。設定中可切換「清晰可讀模式」、調整折射強度與透明度。
- **整面互動地圖背景**：地圖鋪滿整個畫面可拖拉縮放，玻璃面板浮在其上。標記會自動偏移，避開左側（電腦）或上方（手機）的面板，保持在可見區。
- **明暗主題 + 地圖樣式**：UI 可選 自動／深色／淺色；地圖可選 自動／深色／淺色／Google 配色。「自動」會跟隨系統深淺色即時切換。
- 電腦：地圖鋪滿背景、面板浮在左側。手機：上方為可捲動的面板區、下方固定一塊地圖，兩者不重疊，地圖不被卡片擋住。
- 整頁鎖定縮放（雙指/雙擊不會放大網頁），但地圖本身的縮放保留。

> **關於折射與瀏覽器**：折射透鏡用 SVG 位移濾鏡（`backdrop-filter: url(#liquid-lens)`）。這是 WebKit 的已知限制——`backdrop-filter` 套 `feDisplacementMap` 在 Safari／iOS 與 Firefox 上無效（WebKit Bugzilla #245510，至今未修），**只有 Chromium 系（Chrome、Edge）會真正渲染折射**。程式載入時偵測瀏覽器：能用的套折射，Safari／Firefox 自動降級為乾淨的模糊玻璃＋光邊＋細微反光，不會破版。這非本程式可繞過，是引擎層級的限制。手機傾斜反光需在第一次觸碰時授予動態權限。

## 演算法說明
地圖碼採三層網格切分：zone → block(30×30) → unit(30×30) → core(10×10)。
編碼／反查邏輯均已驗證，城市座標 round-trip 誤差 < 2 公尺。

## 關於 Google 搜尋
純前端無法安全內嵌 Google API 金鑰，故 Google 模式以「開啟 Google 地圖搜尋頁」方式呈現；
若日後要在程式內直接取得 Google 座標，需自建後端代理金鑰。

## 公開部署前須知（重要）

本專案若公開供他人使用，請注意以下事項：

**商標**：「マップコード（MAPCODE）」為株式会社 DENSO 之註冊商標。介面已避免以商標作為品牌名稱，並於頁尾標註與 DENSO 無關聯之免責聲明。請勿移除該聲明，亦勿暗示與 DENSO 有官方關係。

**演算法與資料**：代碼計算邏輯與 zone 邊界座標係逆向工程取得。逆向工程之合法性各法域不一，且座標資料可能涉及原始資料權利。非商業、開源用途風險較低，但若商業化，建議先諮詢智慧財產權律師。

**第三方服務流量限制**：
- **Nominatim**（地名搜尋）：OSM 官方使用政策限制每秒最多 1 次請求，且不得用於大量／重度用途。公開後若多人同時使用極易違反，甚至遭封鎖 IP。正式公開建議自架 Nominatim 或改用商業地理編碼服務。
- **CARTO 圖磚**：免費方案有用量上限，流量大可能受限或需付費 API key。

**出處標註**：OSM 資料採 ODbL 授權，必須保留出處標註（頁尾已含）。

> 以上為一般性風險資訊，非法律意見。商業化或需完全確定法律狀態，請諮詢專業律師。
